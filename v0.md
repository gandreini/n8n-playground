# n8n Prototype Playground

Shared prototyping playground for the n8n design team. One Next.js app, every designer gets a namespace.

## Tech stack

- Next.js 16 (App Router), React 19, TypeScript
- **styled-jsx** for all our own component styling (scoped CSS co-located with the component)
- n8n design tokens as CSS custom properties in `app/globals.css`
- shadcn/ui component library (vendored in `components/shadcn/`; hands-off)
- Tailwind CSS 4 is installed to power Shadcn but **not used in our own code**
- pnpm

## Structure

Prototypes go in `app/prototypes/{username}/{prototype-name}/`:

```
app/prototypes/
├── giulio/
│   └── workflow-editor/
│       ├── page.tsx
│       └── metadata.json
└── _templates/
    ├── blank/
    └── n8n-app/
```

## Rules

### Always confirm the active prototype

Before making any changes, you must know which prototype you're working on. If you don't know or there's any ambiguity, ask the user:

> "Which prototype are you working on?"

Never assume. If context was lost, ask again.

### General rules

- Work in your own directory: `app/prototypes/{your-username}/`
- Don't modify other people's prototypes
- Don't modify root files (layout.tsx, page.tsx, globals.css)
- Each prototype is standalone

## Available components

- `components/n8n/prototype-shell.tsx` — n8n sidebar chrome wrapper (use this for n8n-style prototypes)
- `components/n8n/sidebar.tsx` — n8n sidebar
- `components/shadcn/` — Full shadcn/ui library (button, dialog, tabs, input, etc.)

## Creating a prototype

1. Create `app/prototypes/{username}/{name}/`
2. Add `metadata.json`:
   ```json
   {
     "title": "My Prototype",
     "description": "What it does",
     "author": "username",
     "date": "2026-04-12",
     "externalUrl": null,
     "featured": false,
     "template": false
   }
   ```
3. Add `page.tsx` with your prototype code

## Conventions

- Use shared components — don't recreate what already exists
- **Write styles with styled-jsx** (each component ends with a `<style jsx>{...}</style>` block). Do **not** write Tailwind utility classes in your own code.
- Reference n8n design tokens from `globals.css` directly in your CSS: `var(--color--orange-500)`, `var(--spacing--md)`, `var(--font-size--sm)`, etc.
- Every prototype needs a `metadata.json`

## Visual verification (MANDATORY)

**Always visually check your work after making UI changes.** Do not report changes as done without verifying them in the browser first.

After implementing a change:
1. Make sure the dev server is running (`pnpm dev`)
2. Open the relevant page in a browser preview
3. Check that the layout, colors, spacing, and content render correctly
4. If something looks wrong, fix it and check again

Never say "it should work" — verify that it does.
