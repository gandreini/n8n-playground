# n8n Prototype Playground

Shared prototyping playground for the n8n design team. One Next.js app, every designer gets a namespace.

## Tech stack

- Next.js 16 (App Router), React 19, TypeScript
- Tailwind CSS 4 with n8n design tokens
- shadcn/ui component library
- pnpm

## Structure

Prototypes go in `app/prototypes/{username}/{prototype-name}/`:

```
app/prototypes/
├── giulio/
│   └── workflow-editor/
│       ├── page.tsx
│       └── metadata.json
└── _templates/
    ├── blank/
    └── n8n-app/
```

## Rules

- Work in your own directory: `app/prototypes/{your-username}/`
- Don't modify other people's prototypes
- Don't modify root files (layout.tsx, page.tsx, globals.css)
- Each prototype is standalone

## Available components

- `components/n8n/prototype-shell.tsx` — n8n sidebar chrome wrapper (use this for n8n-style prototypes)
- `components/n8n/sidebar.tsx` — n8n sidebar
- `components/ui/` — Full shadcn/ui library (button, dialog, tabs, input, etc.)

## Creating a prototype

1. Create `app/prototypes/{username}/{name}/`
2. Add `metadata.json`:
   ```json
   {
     "title": "My Prototype",
     "description": "What it does",
     "author": "username",
     "date": "2026-04-12",
     "externalUrl": null,
     "featured": false,
     "template": false
   }
   ```
3. Add `page.tsx` with your prototype code

## Conventions

- Use shared components — don't recreate what already exists
- Use Tailwind classes and n8n design tokens from `globals.css`
- Every prototype needs a `metadata.json`
